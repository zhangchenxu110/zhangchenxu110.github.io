1. 环境

OS: Ubuntu 16.04 LTS

name ip 角色
master 10.66.3.31 master
node1 10.66.3.7 slave 1
node2 10.66.3.44 slave 2

Hadoop 版本： 2.7.1
Spark 版本：2.0.0
Java 版本：Oracle jdk 1.8.0_131

2. 前期准备

安装 JDK

Ubuntu 安装 java

安装SSH

Ubuntu 安装 SSH

修改服务器的主机名和hosts文件

  • 修改主机名, 分别在每个机器上执行
sudo hostname master
sudo hostname node1
sudo hostname node2
  • 修改hosts文件
sudo vi /etc/hosts

# 在末尾追加
10.66.3.31  master
10.66.3.44  node2
10.66.3.7   node1

修改成功后, 可以直接通过名字ping通

更改为统一系统时区, 确保所有节点的时间一致精确到秒

sudo tzselect

根据提示选择 Asia China Beijing Time yes (在master上)
然后将Asia/Shanghai shell scripts 复制到/etc/localtime

sudo cp /usr/share/zoneinfo/Asia/Shanghai  /etc/localtime

配置master免密钥SSH登陆所有node

  • 先安装这个(新版的ssh貌似会自动安装)
sudo apt install rsync
  • 生成pub公钥( 在 ~/.ssh/ 目录下)
#一路回车就好
ssh-keygen -t rsa
  • 将公钥发放到其他slave下(在master中执行)
ssh-copy-id -i ~/.ssh/id_rsa.pub master
ssh-copy-id -i ~/.ssh/id_rsa.pub node1
ssh-copy-id -i ~/.ssh/id_rsa.pub node2

安装后之后, 就可以直接ssh无密码登陆到每一个节点

3. 配置完全分布式Hadoop集群

先在master上面改好所有的配置文件，然后将Hadoop包用scp复制到所有子节点
scp 跨机远程copy

各节点分工

解压Hadoop

在所有节点中放入Hadoop, 并解压到 /opt 目录下

配置 hadoop 环境变量和启动脚本

sudo vim /etc/profile.d/hadoop2.7.1.sh

写入:

#!/bin/sh
# Path：/etc/profile.d/hadoop2.7.1.sh

export HADOOP_HOME="/opt/hadoop-2.7.1"
export PATH="$HADOOP_HOME/bin:$PATH"
export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop
export YARN_CONF_DIR=$HADOOP_HOME/etc/hadoop

配置 hadoop-env.sh

所有的配置文件位于 /opt/hadoop-2.7.1/etc/hadoop/

添加JAVA_HOME(必须要配置，否则报错)

export JAVA_HOME=/usr/lib/jvm/java-8-oracle

在master上配置slaves文件

追加

master
node1
node2

配置 core-site.xml

先创建好hadoop各种目录
临时目录： mkdir -p ~/hadoop/tmp
namenode目录： mkdir -p ~/hadoop/hdfs/namenode
datanode目录： mkdir -p ~/hadoop/hdfs/datanode

<configuration>
        <!-- 指定hdfs的nameservice为ns1 -->
        <property>
                <name>fs.defaultFS</name>
                <value>hdfs://master:9000</value>
        </property>
        <!-- Size of read/write buffer used in SequenceFiles. -->
        <property>
                <name>io.file.buffer.size</name>
                <value>131072</value>
       </property>
        <!-- 指定hadoop临时目录,自行创建 -->
        <property>
                <name>hadoop.tmp.dir</name>
                <value>/home/cisdi/hadoop/tmp</value>
        </property>
</configuration>

配置 hdfs-site.xml

注意namenode 和 datanode目录

<configuration>
    <property>
      <name>dfs.namenode.secondary.http-address</name>
      <value>master:50090</value>
    </property>
    <property>
      <name>dfs.replication</name>
      <value>2</value>
    </property>
    <property>
      <!-- 指定namenode 目录，自行创建 -->
      <name>dfs.namenode.name.dir</name>
      <value>file:/home/cisdi/hadoop/hdfs/namenode</value>
    </property>
    <property>
      <!-- 指定datanode目录，自行创建 -->
      <name>dfs.datanode.data.dir</name>
      <value>file:/home/cisdi/hadoop/hdfs/datanode</value>
    </property>
</configuration>

配置 yarn-site.xml

注意修改自己的master名字

<configuration>

<!-- Site specific YARN configuration properties -->

          <value>mapreduce_shuffle</value>
     </property>
     <property>
           <name>yarn.resourcemanager.address</name>
           <value>master:8032</value>
     </property>
     <property>
          <name>yarn.resourcemanager.scheduler.address</name>
          <value>master:8030</value>
      </property>
     <property>
         <name>yarn.resourcemanager.resource-tracker.address</name>
         <value>master:8031</value>
     </property>
     <property>
         <name>yarn.resourcemanager.admin.address</name>
         <value>master:8033</value>
     </property>
     <property>
         <name>yarn.resourcemanager.webapp.address</name>

配置 mapred-site.xml

在2.7.1 版本中， 只有mapred-site.xml.template。记得新建mapred-site.xml 或者 将模板复制到xml中

<configuration>
  <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
  </property>
  <property>
        <name>mapreduce.jobhistory.address</name>
        <value>master:10020</value>
  </property>
  <property>
        <name>mapreduce.jobhistory.address</name>
        <value>master:19888</value>
  </property>
</configuration>

5.解决bug

a. 用户权限问题

在测试运行时， 一直提示不允许执行 permission denied
方法：修改hadoop安装文件夹的权限为当前用户

sudo chown -R cisdi:cisdi hadoop-2.7.1

注意用户和用户组

b. 配置文件找不到

将 /opt/hadoop-2.7.1/etc/hadoop 下的所有文件修改为可执行权限

chmod 777 /opt/hadoop-2.7.1/etc/hadoop
chmod 777 /opt/hadoop-2.7.1/etc/hadoop/*
