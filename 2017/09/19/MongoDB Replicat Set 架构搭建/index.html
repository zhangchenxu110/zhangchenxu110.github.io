1. 架构解析

ReplicatSet结构如下


Primary Node能够将自己的数据同步到Secondary Node, 以实现数据的冗余备份.
一旦primary Node挂了，或者10s内无法取得Heartbeat，Secondary Node会选举新的Primary Node

正常状态

Primary 宕机

2. 配置

首先在机器上面部署好MongoDB服务 我的笔记-NoSQL-MongoDB单机部署

这里我们用单机开启四个MongoDB进程

Node Num hosts
Primary Node 1 node2:28001
Secondary Node 2 node2:28002
node2:28003
Arbiter Node 1 node2:28004

建立数据和日志文件夹

# 数据目录
mkdir -p /mongodb/data/1
mkdir -p /mongodb/data/2
mkdir -p /mongodb/data/3
mkdir -p /mongodb/data/4
# 日志目录
mkdir -p /mongodb/log/1
mkdir -p /mongodb/log/2
mkdir -p /mongodb/log/3
mkdir -p /mongodb/log/4
# 配置文件目录
mkdir -p /mongodb/conf

目录结构如下:

mongodb3
├── conf
├── data
│   ├── 1
│   ├── 2
│   ├── 3
│   └── 4
└── log
    ├── 1
    ├── 2
    ├── 3
    └── 4

修改配置文件(需要写四个)

样本 [1.yaml]:

# 1.yaml
# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/

# Where and how to store data.
storage:
  dbPath: /mongodb/data
  journal:
    enabled: true
#  engine:
#  mmapv1:
#  wiredTiger:

# where to write logging data.
systemLog:
  destination: file
  logAppend: true
  path: /mongodb/log/mongodb.log 

# network interfaces
net:
  port: 28001
 # bindIp: 127.0.0.1

processManagement:
  fork: true
#security:
#operationProfiling:

# 副本
replication:
  replSetName: DBTEST
#sharding:
## Enterprise-Only Options:
#auditLog:
#snmp:

每个配置文件要修改dbpath, logpath, 还有port就好

3. 启动

mongod -f 1.yaml
mongod -f 2.yaml
mongod -f 3.yaml
mongod -f 4.yaml

用mongo连接任意一个端口

mongo --port 28001

在mongo shell中配置副本

#Initiate the rplica set
rs.initiate()
#display the replica set configuration object
rs.conf()
#Add the remaining members to the replica set
rs.add("node2:28002")
rs.add("node2:28003")
rs.addArb("node2:28004") #返回{ "ok" : 1 }
